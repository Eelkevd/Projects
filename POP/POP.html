PDP

Under the PDP tab you will find my personal development plan. I have divided this plan into four different phases:

  • Orientation phase

    In the orientation phase I will mostly gather information on what programming language are best suited for my needs. A healthy combination of dreaming and pragmatism are key here. The goal is to try a lot of different programming languages and see which one I like. Also, now is the time to start oriÃ«nting on which programming languages are in demand, so that I can focus my efforts on a language that put me in a favorable position on the market. Anything is still possible in this phase.

  • Development phase

    In the development phase I will start focusing my efforts, while still keeping some room for peripheral activities. From the orientation phase I have distilled an aim but it's not set in stone. In this phase I will do lots of small projects and challenges so that I can expand my programming knowledge quickly and practically. I personally believe a hands-on approach is the best approach when trying to learn programming.

  • Specialization phase

    In the specialization phase I double down on a very specific list of skills that I want to learn. From the development phase I now have a good foundation on which I can build extensive programming knowledge. In this phase I will start reading books and texts to supplement my experience. I will have a clear goal and I know what I will have to learn to get where I want to be. At the same time I will take on a big project as a crowning piece on my portfolio.

  • Dream phase

    The dream phase is a long term goal that may or may not be fulfilled in the future. I believe it is good to have high goals, because they give you something to work towards. At the same time, you must not take these goals so seriously that they'll demotivate you if you can't attain them. For this section I asked my self the question "if there were no limitations, what would I want to do?" Then, whenever I am faced with a fork in the road, I can make the decision that is most in line with my dream goal.

Personal webpage

I built a personal webpage that contains a personal development plan and a portfolio, you're looking at it now! With this project I learned that my aptitudes include coding more than design. I had a blast getting the interactivity and the framework to work, but struggled with making things pretty. I also started adding comments to my javascript, because they easily become a mess if you don't.

Address book

I built an address book with a local storage for my next exercise. The focus of this exercise was mainly on Javascript on there was a lot to be learned. The mark of a good project is that feeling of "I would have done things entirely differently if I had to do it again", that means you learned something. With this project I had that feeling very strongly.

The address book functions exactly as it should, but the code is a little clumsy in some places. I did not want to use a tutorial, because it was a learning project and I didn't want to copy code blindly. I added all functionality and then the localstorage, which is definitely the wrong way around. Aside from that I think the project was great as a learning experience. My understanding of javascript and writing clean code improved a lot.

Simple calculator

My very first excercise was to build a simple calculator. This was a good way to get acquainted with using HTML, CSS and Javascript. This was a project where I learned that programming often means that debugging takes up a big chunk of your time. I also used one file for my HTML, CSS and javascript. That's something I won't be doing in the future again because when a project gets big it's easy to lose track of where everything is.
